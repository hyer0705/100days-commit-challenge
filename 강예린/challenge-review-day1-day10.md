# 10일 동안 공부한 내용
- DP, Dynamic Programming
    - DP, 즉 다이나믹 프로그래밍은 복잡한 문제를 더 작은 하위 문제로 나누어 해결하는 알고리즘 설계 기법이다.
    - 하향식(Top-down) vs 상향식(Bottom-up)
      - 재귀적 호출은 주로 하향식(top-down) 접근 방식을 사용한다. 즉, 큰 문제를 작은 하위 문제로 나누어 해결하는 방식이다.
      - 반면에 다이나밋 프로그래밍은 주로 상향식(bottom-up) 접근 방식을 사용한다. 작은 하위 문제들부터 시작하여 그 결과를 저장하고, 이를 이용하여 점진적으로 큰 문제의 해를 구해나간다.
    - 메모이제이션(Memoization)
      - DP는 중복되는 계산 결과를 저장하는 메모리 기법인 메모이제이션을 사용한다.
      - 이전에 계산한 값을 캐시하고, 다시 필요할 때 해당 값을 가져와 재사용한다.
      - 재귀적 호출에서의 중복 계산을 방지하고, 계산 속도를 향상시킨다.
